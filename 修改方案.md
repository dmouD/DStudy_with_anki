## 0. 当前问题总结

当前 DStudy 已经可以在 Windows 下完成 CMake 配置、编译、`windeployqt` 部署，并且已经成功生成 NSIS 安装包：

```text
F:/test_qt/StudyPlanner/build-win/DStudy-1.0-win64.exe
```

但是目前还有几个需要正式修复的问题：

```text
1. 安装包运行时缺少 libz.dll
2. Windows 打包后的界面样式和开发环境中不一致
3. Anki 复习页面顶部布局较挤，样式不统一
4. 新安装环境数据库为空，需要确认这是正常现象
5. Windows 安装包需要保证 DStudy.exe、Qt DLL、SQLite 驱动、libz.dll 都被打进去

```

本方案目标：

```text

```

# 1. 修复缺少 libz.dll 的问题

## 1.1 问题原因

当前项目使用了：

```cmake
find_package(ZLIB REQUIRED)
```

并且完整 Anki `.apkg` 导入功能依赖 zlib。Windows 下通过 vcpkg 安装的 zlib 文件实际名称是：

```text
C:/vcpkg/installed/x64-mingw-dynamic/bin/libz.dll
```

不是：

```text
zlib1.dll
```

当前 `windeployqt` 只会自动部署 Qt 相关 DLL，不会自动部署 vcpkg 里的 `libz.dll`，所以安装包中会缺少它。

---

## 1.2 CMakeLists.txt 中增加 libz.dll 安装规则

在 `CMakeLists.txt` 中找到已有的 `install(TARGETS DStudy ...)` 和 CPack 配置区域，在合适位置加入下面代码。

推荐写法：

```cmake
if(WIN32)
    set(DSTUDY_VCPKG_ROOT "C:/vcpkg" CACHE PATH "Path to vcpkg root")
    set(DSTUDY_ZLIB_DLL "${DSTUDY_VCPKG_ROOT}/installed/x64-mingw-dynamic/bin/libz.dll")

    if(EXISTS "${DSTUDY_ZLIB_DLL}")
        install(FILES "${DSTUDY_ZLIB_DLL}"
            DESTINATION bin
        )
        message(STATUS "DStudy: libz.dll will be installed from ${DSTUDY_ZLIB_DLL}")
    else()
        message(WARNING "DStudy: libz.dll not found at ${DSTUDY_ZLIB_DLL}. The installed program may fail to start.")
    endif()
endif()
```

如果你的安装目标不是 `bin`，需要根据实际安装目录调整。但从当前 CPack 输出看，安装包内部路径是：

```text
DStudy-1.0-win64/bin/DStudy.exe
```

所以这里应该使用：

```cmake
DESTINATION bin
```

---

## 1.3 检查安装包中是否包含 libz.dll

重新打包后执行：

```bat
dir F:\test_qt\StudyPlanner\build-win\_CPack_Packages\win64\NSIS\DStudy-1.0-win64\bin\libz.dll
```

如果能看到：

```text
libz.dll
```

说明安装包里已经包含 zlib 运行库。

# 2. 修复 Windows 界面样式不一致问题

## 2.1 问题现象

打包后的 Windows 版本界面出现：

```text
1. 窗口外围偏黑
2. Anki 页面顶部控件显得拥挤
3. 部分控件使用了 Windows 原生默认样式，看起来和主界面不统一
4. 字体、按钮、背景色和开发时显示不完全一致
```

这不是程序坏了，而是 Windows 下 Qt 默认样式、系统深色模式、字体渲染和 Linux/开发环境不同造成的。

---

## 2.2 main.cpp 中统一应用 Fusion 浅色样式

在 `main.cpp` 中找到：

```cpp
QApplication a(argc, argv);
```

在它后面加入：

```cpp
QApplication::setStyle("Fusion");

QPalette palette;
palette.setColor(QPalette::Window, QColor("#f5f7fb"));
palette.setColor(QPalette::WindowText, QColor("#111827"));
palette.setColor(QPalette::Base, QColor("#ffffff"));
palette.setColor(QPalette::AlternateBase, QColor("#f1f5f9"));
palette.setColor(QPalette::Text, QColor("#111827"));
palette.setColor(QPalette::Button, QColor("#ffffff"));
palette.setColor(QPalette::ButtonText, QColor("#111827"));
palette.setColor(QPalette::Highlight, QColor("#2563eb"));
palette.setColor(QPalette::HighlightedText, QColor("#ffffff"));
a.setPalette(palette);
```

如果 `main.cpp` 没有包含相关头文件，需要补充：

```cpp
#include <QApplication>
#include <QPalette>
#include <QColor>
```

作用：

```text
1. 强制使用 Qt Fusion 风格
2. 避免 Windows 深色模式影响应用内容区域
3. 统一浅色背景、文字颜色、按钮颜色
```

---

## 2.3 MainWindow 中统一全局字体和背景

在 `MainWindow` 构造函数中，`setupUi` 或主界面初始化之后加入：

this->setStyleSheet(

注意：如果项目中已经有比较完整的 QSS，不要直接覆盖全部样式，应合并进去。

## 3.3 给 Anki 页面顶部增加边距

检查 `ReviewWidget` 顶层布局，例如：

```cpp
QVBoxLayout *mainLayout = new QVBoxLayout(this);
```

建议设置：

```cpp
mainLayout->setContentsMargins(20, 16, 20, 16);
mainLayout->setSpacing(12);
```

对于顶部标题区域布局，建议设置：

```cpp
headerLayout->setContentsMargins(0, 0, 0, 8);
headerLayout->setSpacing(8);
```

对于牌组卡片区域，建议：

```cpp
deckLayout->setSpacing(12);
```

目标效果：

```text
Anki 页面不再紧贴窗口边缘，顶部文字、搜索框、按钮之间留出合理空隙。
```

# 4. 数据为空问题说明与处理

## 4.1 当前现象

打包安装后打开软件，看到：

```text
任务列表为空
默认牌组 0 张
默认分组 0 张
```

这是正常现象。

原因是 Windows 安装版使用的是新的用户数据目录，不会读取开发目录或 Linux 环境的旧数据库。

---

## 4.2 确认数据库路径使用 QStandardPaths

检查 `DatabaseManager` 中数据库路径是否使用了：

```cpp
QString dataDir = QStandardPaths::writableLocation(QStandardPaths::AppDataLocation);
QDir().mkpath(dataDir);
QString dbPath = dataDir + "/dstudy.db";
```

Windows 下应保存到类似：

```text
C:/Users/用户名/AppData/Roaming/DStudy/dstudy.db
```

不要把数据库写到程序安装目录，例如：

```text
C:/Program Files/DStudy
```

因为普通用户没有写权限。

---

## 4.3 建议在调试输出中打印数据库路径

在数据库初始化时临时加入：

```cpp
qDebug() << "DStudy database path:" << dbPath;
```

方便确认安装包运行时读写的是哪个数据库。

# 8. CMakeLists.txt 推荐最终补丁

下面是建议加入到 `CMakeLists.txt` 的关键内容。

## 8.1 保证 Windows 安装 libz.dll

```cmake
if(WIN32)
    set(DSTUDY_VCPKG_ROOT "C:/vcpkg" CACHE PATH "Path to vcpkg root")
    set(DSTUDY_ZLIB_DLL "${DSTUDY_VCPKG_ROOT}/installed/x64-mingw-dynamic/bin/libz.dll")

    if(EXISTS "${DSTUDY_ZLIB_DLL}")
        install(FILES "${DSTUDY_ZLIB_DLL}"
            DESTINATION bin
        )
        message(STATUS "DStudy: libz.dll will be installed from ${DSTUDY_ZLIB_DLL}")
    else()
        message(WARNING "DStudy: libz.dll not found at ${DSTUDY_ZLIB_DLL}")
    endif()
endif()
```

## 8.2 确保 Windows 不弹出黑色控制台

```cmake
if(WIN32)
    set_target_properties(DStudy PROPERTIES
        WIN32_EXECUTABLE TRUE
    )
endif()
```

## 8.3 确保安装目标正确

```cmake
include(GNUInstallDirs)

install(TARGETS DStudy
    RUNTIME DESTINATION bin
    BUNDLE DESTINATION .
)
```

# 9. main.cpp 推荐最终补丁

```cpp
#include <QApplication>
#include <QPalette>
#include <QColor>

int main(int argc, char *argv[])
{
    QApplication a(argc, argv);

    QApplication::setStyle("Fusion");

    QPalette palette;
    palette.setColor(QPalette::Window, QColor("#f5f7fb"));
    palette.setColor(QPalette::WindowText, QColor("#111827"));
    palette.setColor(QPalette::Base, QColor("#ffffff"));
    palette.setColor(QPalette::AlternateBase, QColor("#f1f5f9"));
    palette.setColor(QPalette::Text, QColor("#111827"));
    palette.setColor(QPalette::Button, QColor("#ffffff"));
    palette.setColor(QPalette::ButtonText, QColor("#111827"));
    palette.setColor(QPalette::Highlight, QColor("#2563eb"));
    palette.setColor(QPalette::HighlightedText, QColor("#ffffff"));
    a.setPalette(palette);

    MainWindow w;
    w.show();

    return a.exec();
}
```

如果你的 `main.cpp` 已有其它初始化代码，不要直接覆盖，按需合并。
